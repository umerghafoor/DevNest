# DevNest plans

Two-stage plan for building DevNest.

| File                                           | Purpose                                                             |
| ---------------------------------------------- | ------------------------------------------------------------------- |
| [01-high-level-plan.md](01-high-level-plan.md) | Phases, goals, exit criteria, architecture, risks. Read this first. |
| [02-low-level-todo.md](02-low-level-todo.md)   | Granular checkbox TODO. Work top-to-bottom.                         |

## How to use

1. Read `01-high-level-plan.md` end to end before starting any phase.
2. For day-to-day work, open `02-low-level-todo.md` and pick the next unchecked item in the current phase.
3. When a phase's exit criteria are met, move to the next phase — don't skip ahead.
4. Update the TODO checkboxes as you go. If a task spawns subtasks bigger than ~30 minutes, add them as nested items.
